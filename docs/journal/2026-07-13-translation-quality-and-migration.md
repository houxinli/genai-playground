# 2026-07-13 翻译质量修复潮 + 旧流水线迁移(gh-142 系列)

## 背景

用户实读发布产物,连续发现三类质量问题,一路排查牵出多个协议缺陷,并顺势完成了 6 个老作者
从旧流水线到新 Artifact Store 的迁移。本条覆盖 PR #143–#153(除 #147/#148 ytmusic)。

## 发现的问题与根因

1. **译文整块错位(block-paste / shift)**:Cursor+Grok 分批翻译时丢失段号同步,把上一批的
   输出重新吐出(26367355:段301-320 = 段271-290)。逐段 QA 只看单段形状,抓不到跨段绑定错。
2. **压缩体断代**:后期 loop 在固定 100 段/批的输出压力下把译文压成电报体(译/源长度比 0.5x,
   17 篇),质量低于阅读器自带机翻。同一模型不同时期,证明是 prompt/harness 设计问题非模型问题。
3. **人名/拟声词假名残留**:job 的 `context_pack.entities` 一直为空——实体库从未接入 agent
   主路线(prepare/finish),人名全靠模型当场发挥,跨篇不一致,7239 行残留假名。
4. **发布带缺口**:填空 TSV 的空行被组装成空候选,经 reviewable 放宽路径发布,212 篇残缺书上线。
5. **finish 遇已有 ref 永不推进**:修复只更新 rendered、store ref 长期指向旧坏版本,verify 看不出。
