# 2026-05-20 死代码清理与开发规范收敛

## 背景

对翻译子系统做了一次结构性体检（83 个 Python 文件、约 1.85 万行），目的是判断是否需要大重构。结论是：**不需要大重构**，但存在明显的死代码与文档分散问题。这轮先清理纯删除项，再把分散的开发规范收敛到单一真相源。

## ✅ 完成工作

### 主要功能

1. **删除 `enhanced_mode` 整条链路**。所有发布 preset 均为 `bilingual_simple=true`，`enhanced_mode` 无任何 caller，是死路径。
2. **删除 `Translator.translate_text` 死分支**。其唯一 caller 引用了不存在的 `config.bilingual` 字段，属 broken-and-dead；非 `bilingual_simple` 路径统一收敛到 `translate_body_text`。
3. **删除 `utils/validation` 下的孤儿校验器**（`quality_validator` / `content_validator`），仅有顶层 re-export，无真实调用方。
4. **开发规范收敛**：把 `AGENTS.md` 扩展为开发规范真相源；瘦身 `docs/AGENT_CONTEXT.md`（去重复/过时内容，只留稳定环境与排障）；更新两处 README 入口。

### 问题解决

- **问题**：`pipeline.py` 看似 god file（1379 行），怀疑要大拆分。
  **判断**：其中约 340 行是 `enhanced_mode` 死路径。删除后回落到合理体量，P2 的阶段化拆分不必现在做。
- **问题**：非 `bilingual_simple` 路径运行即 `AttributeError`。
  **方案**：该分支本就走不到（生产全走 simple），删除后由 `translate_body_text` 兜底，反而恢复成应有行为。
- **问题**：规范散落在 `AGENTS.md`（过简）、`AGENT_CONTEXT.md`（混入成果罗列）、各 README，新 agent 难定位。
  **方案**：单一真相源 `AGENTS.md`，其余文档指向它。
